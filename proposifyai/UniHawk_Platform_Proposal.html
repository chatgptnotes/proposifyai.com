Executive Summary

Project Overview

UniHawk requires a proprietary EdTech platform to replace Wise Classroom and extend capabilities across tutoring, test preparation, and counselling/admissions workflows. The platform must support multi-country, multi-branch operations with strong governance, while maintaining the flexibility to white-label for future commercialization.

Strategic Objectives

  • Operational Independence: Own the product roadmap, UX, and feature development
  • Unified Platform: Consolidate tutoring, test prep, and counselling into one system
  • Multi-Tenant Foundation: Support current operations + future white-label clients
  • Global Scale: Multi-country, multi-branch governance from day one
  • Future-Ready: Modular architecture enables SaaS commercialization

Solution Approach

We propose a three-phase delivery model, with Phase 1 focused on achieving feature parity with Wise Classroom while introducing UniHawk-specific requirements (multi-branch governance, credits, parent billing, and counselling basics).

1
Phase 1: Foundation
Wise replacement + UniHawk essentials. Target: 20 weeks development and pilot rollout before July 2026 academic year.

System Architecture Principles

1. Multi-Tenant Foundation

  • Tenant Isolation: Strict data segregation with PostgreSQL row-level security
  • Global Identity: Users can belong to multiple tenants without duplication
  • Org Hierarchy: Country → Branch → Programme model enforced at database level
  • Role Scoping: Permissions are tenant + org-scoped (e.g., Branch Admin for Branch A only)

2. Modular Design

Platform delivered as independent, composable modules:

  • Classroom Module: Live sessions, recordings, artefacts, analytics
  • Learning Module: Timeline, content, assignments, submissions, discussions
  • Test Module: Create, assign, grade, reuse tests across cohorts
  • Counselling Module: Meetings, notes, milestones, document tracking
  • Billing Module: Credits, invoices, payments, parent portal
  • Integrations Module: Zoom, email, WhatsApp, push notifications, CRM/accounting

3. API-First Architecture

  • RESTful APIs: All core operations exposed via documented REST endpoints
  • Webhook System: Event-driven integrations (meeting lifecycle, billing, user management)
  • Provider Abstraction: Swap CRM, accounting, WhatsApp, payment providers without refactoring
  • Rate Limiting: Per-tenant API quotas to prevent abuse

4. Security & Compliance

  • Authentication: OAuth 2.0, magic links, session management, audit trails
  • Authorization: Role-based access control (RBAC) with least-privilege enforcement
  • Data Encryption: At-rest (AES-256) and in-transit (TLS 1.3)
  • Privacy Controls: Recording access restrictions, download gating, parent/student separation
  • Audit Logging: All admin actions, role changes, sensitive operations logged
Phase 1: Detailed Scope
Page 4

Module 1: Identity, Tenancy & Governance

Features

  • Multi-tenant architecture with strict data isolation
  • Global user identity (one account across multiple tenants)
  • Org hierarchy: Country → Branch → Programme
  • Role-based permissions: Super Admin, Country Admin, Branch Admin, Tutor, Counsellor, Student, Parent, Finance Manager
  • Tutor affiliation model: Home Country/Branch + cross-branch/cross-country assignments
  • Audit trails for admin actions and sensitive operations

Acceptance Criteria

  • No cross-tenant data visibility (validated via penetration testing)
  • Country Admin sees only their country; Branch Admin sees only their branch
  • Only Super Admin can create Countries and appoint Country Admins
  • Tutors assigned across branches/countries see only their assigned cohorts

Module 2: Classroom Delivery (Zoom Integration)

Features

  • Zoom meeting creation (single + recurring sessions)
  • Meeting lifecycle capture (started, ended, participant join/leave times)
  • Join gating: registered email/unique ID enforcement (time-bound exceptions supported)
  • Recording retrieval (link + metadata) with retry logic
  • Automated transcript generation (time-stamped, Whisper API)
  • AI session summary (key topics, explanations, homework, flags for tutor confirmation)
  • Recording access controls:
    • Students can stream recordings (no downloads, no parent streaming)
    • Only Country Admin and Super Admin can download recordings + transcripts
  • Video processing: adaptive streaming, high quality at low file size
  • Post-session tutor wrap-up: topic covered + comments (structured fields)
  • Credit consumption triggers on session completion

Acceptance Criteria

  • ≥95% sessions produce recording link + artefacts within agreed SLA
  • Join flow p95 ≤ 2 seconds from "Join" click to Zoom link availability
  • Transcript + AI summary available for ≥90% of recorded sessions (where enabled)
  • Download attempts by students are blocked (UI + direct link protection)

Module 3: Courses, Cohorts & Scheduling

Features

  • Create/manage courses and cohorts (group tutoring/test prep)
  • Create/manage 1:1 meeting types (tutoring and counselling)
  • Student enrolment and tutor assignment
  • Scheduling: single + recurring sessions with timezone support
  • Staff scheduling: Admins, tutors, counsellors can schedule (role-based permissions)
  • Student scheduling requests:
    • Submit preferred slots + reason/topic + programme
    • Credit validation before submission
    • Routes to appropriate staff role for approval
  • Configurable terminology (Class vs Meeting vs Session)
  • Student and tutor calendar views

Acceptance Criteria

  • Branch Admin can create cohort, assign tutor, enrol students, schedule recurring sessions
  • Students see upcoming + past sessions for enrolled cohorts
  • Tutors see timetable across all assignments; one-click join
  • Students can submit scheduling requests only with sufficient credits and only for enrolled programmes

Module 4: Learning Feed (Timeline, Content, Submissions)

Features

  • Timeline posts: text, file upload, video link, voice note
  • Comment controls per post (enable/disable)
  • Student uploads/submissions for assignments and homework
  • Version tracking for submissions
  • Moderation: hide/delete content (authorised roles)
  • Responsive web + mobile access

Acceptance Criteria

  • Tutor can post to cohort timeline; admin can disable comments
  • Students can view timeline content on web and mobile
  • Students can submit homework with timestamp and status visible to tutor

Module 5: Assignments & Assessments

Features

  • Create assignments with due dates and submission tracking
  • Create tests/quizzes with multiple question types
  • Assign tests to batches/cohorts
  • Test reuse: within-country by default; cross-country via Global Test Library (Super Admin curated)
  • Basic grading and results publishing
  • Test categorization for smooth retrieval

Acceptance Criteria

  • Tutor can create assignment with due date; see submission status per student
  • Tutor can publish test to cohort; students can view and submit
  • Tests can be reused across branches/countries with proper access controls

Module 6: Analytics (Lens-Style Baseline)

Features

  • Attendance % computed per student per session (from join/leave times)
  • Engagement metrics framework:
    • Attentiveness score (definition documented and versioned)
    • Talk-time score (teacher vs student proportions)
  • Session report view for tutors and admins
  • Rollups by cohort, branch, country, and date range
  • Export capabilities for custom reporting

Acceptance Criteria

  • Attendance % visible on session report for all participants
  • Branch and Country Admin dashboards show attendance rollups
  • Engagement metrics versioned to preserve historical reporting accuracy

Module 7: Billing, Credits & Parent Portal

Features

  • Credit ledger per student: purchased, consumed, remaining
  • Credit enforcement rules:
    • Prevent/flag scheduling and joining when insufficient
    • Low-credit alerts to parents
    • Immediate reinstatement post top-up
  • Payment gateway integration (Stripe first; abstraction for regional gateways)
  • Parent portal: credit balance, top-up, invoice/receipt view
  • Finance separation:
    • Finance Manager: transaction amounts + reconciliation
    • Admins: credit counts + depletion (no transaction amounts)

Acceptance Criteria

  • ≥99.5% credit consumption accuracy across all pilot sessions
  • ≥98% payment success rate (excluding bank/network outages)
  • <0.5% sessions require manual billing correction

Module 8: Notifications & Reporting

Features

  • Email notifications (automatic):
    • Enrolment welcome/intro
    • Session reminders (24h/1h before, timezone-aware)
    • Low-credit alerts
    • Weekly/monthly family reports
  • WhatsApp notifications (provider-agnostic, automatic):
    • Enrolment confirmations
    • Session reminders
    • Low-credit alerts
  • Push notifications (mobile):
    • Session reminders
    • Low-credit alerts
  • Sending domain configuration (SPF/DKIM/DMARC)
  • Automated + manual report triggers

Acceptance Criteria

  • Enrolment notifications sent within agreed SLA
  • Session reminders trigger based on configured lead times; respect timezones
  • Weekly/monthly reports delivered on schedule; manual triggers supported
  • Email deliverability passes smoke test (no major provider blocks)

Module 9: Integrations, APIs & Webhooks

Features

  • REST API for core entities: users, courses, sessions, payments
  • API documentation (routes, auth, rate limits)
  • Token-based auth with least-privilege scopes
  • Webhook events:
    • Meeting started/ended
    • Recording completed
    • Attendance computed
    • Invoice/payment events
    • Student/teacher added/removed
  • Webhook security: signed payloads, retry + backoff, dead-letter queue
  • Webhook payload versioning and idempotency (event_id + timestamp)
  • Integration adapters: CRM/accounting (Zoho first; vendor-agnostic)

Acceptance Criteria

  • API endpoints documented and secured
  • ≥99% webhook delivery success rate (with retry)
  • For ≥95% of sessions: meeting ended → recording → attendance → credit update observable end-to-end
  • Providers (CRM, WhatsApp, payments) swappable without core refactoring

Security Architecture

  • Authentication: Supabase Auth (OAuth 2.0, magic links, session tokens)
  • Authorization: Role-based access control (RBAC) with PostgreSQL row-level security (RLS)
  • Encryption:
    • At-rest: AES-256 encryption for database and file storage
    • In-transit: TLS 1.3 for all API communications
  • API Security:
    • JWT token-based authentication
    • Rate limiting (per-tenant quotas)
    • CORS configuration
    • Input validation and sanitization
  • Audit Logging: All admin actions, role changes, sensitive operations logged with timestamps
  • Recording Access:
    • Signed URLs with expiration (short-lived tokens)
    • IP-based restrictions for admin downloads
    • Download gating (students cannot download)

Infrastructure & DevOps

  • Hosting: Vercel (web) with global CDN and auto-scaling
  • Database: Supabase (PostgreSQL) with automatic backups and point-in-time recovery
  • Storage: Supabase Storage + AWS S3 for recordings and documents
  • CI/CD: GitHub Actions for automated testing and deployment
  • Monitoring:
    • Sentry for error tracking and performance monitoring
    • LogRocket for session replay and user behavior analytics
    • Posthog for product analytics and feature usage
  • Alerting: PagerDuty integration for critical incidents (session failures, payment issues)
  • Backups:
    • Database: Daily automated backups with 30-day retention
    • Recordings: Redundant storage across multiple regions
